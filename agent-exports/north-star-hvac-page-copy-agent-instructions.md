## 12. Template-Fit Rules

The selected page template/spec controls the output.

Match:

- Section order
- Section IDs
- Field names
- Number of cards/items/steps/FAQs
- Button-label fields
- Navigation/footer link formats
- Visible copy capacity

Do not output extra labels inside the rendered copy. The bulk paste section should only contain section IDs and field values.

Use this format for bulk copy:

```txt
### 01-section-id
fieldName: Copy value
fieldName: Copy value

### 02-section-id
fieldName: Copy value
repeatableItems: Item title - item description
Second item title - second item description
```

Repeatable item rules:

- One item per line.
- If an item has a title and description, use `Title - Description`.
- Do not split one item across multiple lines.
- Do not use commas or semicolons to separate multiple items.
- For mixed-size grid cards, prefix larger items with `[large]` or `[featured]` only when the template/spec says the layout needs that marker.

Button rules:

- `primaryAction` and `secondaryAction` should be button labels only unless the template/spec asks for URLs.
- Keep button labels short, usually 10 to 22 characters.
- Use `Request a Quote`, `Call Now`, `View Services`, `Explore Services`, or similarly plain labels.

Navigation/footer rules:

- `navLinks` should use `Label -> /path`.
- `footerLinks` should use `Label -> /path` where possible.
- `legalLine` should contain only the copyright/legal line.

## 13. Character-Fit Rules

If the page template/spec includes example text and character counts, follow those limits.

Default fit rules when no exact limit is supplied:

- Eyebrows: 12 to 36 characters.
- H1 / hero title: 45 to 75 characters unless the template visibly supports more.
- Section headings: 35 to 70 characters.
- Card titles: 18 to 45 characters.
- Card descriptions: 80 to 160 characters.
- Short body paragraphs: 120 to 220 characters.
- Long body fields: 220 to 450 characters, only when the template clearly expects long-form copy.
- Button labels: 10 to 22 characters.
- Navigation labels: 6 to 18 characters.
- FAQ questions: 45 to 95 characters.
- FAQ answers: 120 to 240 characters.

Prefer target ranges over exact character matching. Never exceed a hard maximum if the template/spec provides one.

If a field needs to be shorter to preserve layout, shorten the copy rather than changing the layout.
